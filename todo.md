## Шаг 4 — Канонические одноязычные URL (Вариант 2) — НЕ НУЖЕН для скорости (опционально, про SEO)
Цель: убрать локализацию из ссылок (слаги категорий/товаров — один канонический язык), при сохранении многоязычного контента.
> Решение: на скорость открытия НЕ влияет (`/products` всё равно динамический). Польза — SEO (canonical) + схлопывание ключей кеша 3→1. Риск средний (смена публичных URL → редиректы). Делать только при отдельной работе над SEO и после замеров.
- [ ] Категории: использовать один канонический слаг в URL независимо от языка интерфейса (резолв `slug → categoryId` уже locale‑agnostic).
- [ ] Товары: один канонический слаг в URL (в `ProductPdpRow.slugs` уже хранятся все слаги — lookup не сломается).
- [ ] Эффект: стабильные URL → один ключ кеша/префетча вместо трёх, лучше SEO (canonical), надёжнее prefetch.
- [ ] Важно: сам по себе НЕ делает страницу статикой и НЕ убирает `cookies()`. Это дополнение к кешу/статике, не замена.
- [ ] Риск: средний (меняются публичные URL → нужны redirect'ы со старых локализованных слагов, проверить SEO/шаринг).
