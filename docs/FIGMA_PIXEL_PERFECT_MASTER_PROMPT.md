## ШАГ 1 — ПОЛУЧИТЬ ДИЗАЙН ИЗ FIGMA

1. Извлеки `fileKey` и `nodeId` из URL.
2. Вызови `get_design_context` с этими параметрами.
3. Из ответа сохрани ВСЁ:
   - **Код** (JSX + Tailwind классы) — это ГЛАВНЫЙ источник правды
   - **Скриншот** — это ВИЗУАЛЬНЫЙ эталон для сравнения
   - **Подсказки** (hints) — прочитай, но НЕ применяй если они противоречат коду
   - **URL ассетов** (картинки, иконки) — используй AS-IS

## ШАГ 2 — РЕАЛИЗАЦИЯ (СТРОГИЕ ПРАВИЛА)

**ЗОЛОТОЕ ПРАВИЛО: код из get_design_context = закон. Не адаптируй, не улучшай, не упрощай.**

### 2.1 — Что брать из Figma КАК ЕСТЬ (НИКОГДА не менять)

- ❌ НЕ заменяй цвета (`#f6f6f6` → НЕ `bg-surface-default`, оставь `bg-[#f6f6f6]`)
- ❌ НЕ заменяй размеры (`w-[306.418px]` → НЕ `w-80`, оставь `w-[306.418px]`)
- ❌ НЕ заменяй отступы (`top-[17px]` → НЕ `top-4`, оставь `top-[17px]`)
- ❌ НЕ заменяй border-radius (`rounded-[32px]` → НЕ `rounded-3xl`, оставь `rounded-[32px]`)
- ❌ НЕ заменяй font-size (`text-[14px]` → НЕ `text-sm`, оставь `text-[14px]`)
- ❌ НЕ заменяй line-height (`leading-[20px]` → НЕ `leading-5`, оставь `leading-[20px]`)
- ❌ НЕ заменяй tracking/letter-spacing
- ❌ НЕ заменяй позиционирование (absolute, inset, left/right/top/bottom %)
- ❌ НЕ заменяй font-family (оставь как в Figma, подключи через CSS variable проекта)
- ❌ НЕ меняй порядок и вложенность элементов
- ❌ НЕ убирай элементы которые кажутся лишними
- ❌ НЕ добавляй элементы которых нет в дизайне

### 2.2 — Что МОЖНО адаптировать (минимально)

- ✅ `<img>` → `<Image>` из `next/image` (с `unoptimized`)
- ✅ `<a>` → `<Link>` из `next/link`
- ✅ Добавить `'use client'` если нужен
- ✅ Добавить `alt=""` к картинкам
- ✅ URL ассетов — вставить прямо в файл как константы (НЕ в отдельный файл)
- ✅ Обернуть в именованный export (`export function ComponentName`)
- ✅ Использовать `font-montserrat-arm` из Tailwind конфига (если шрифт = Montserrat)

```tsx
    // КОД ИЗ GET_DESIGN_CONTEXT С МИНИМАЛЬНЫМИ ПРАВКАМИ
  );

```

### 2.4 — ЗАПРЕЩЕНО

- ❌ Создавать отдельный файл `assets.ts`
- ❌ Использовать дизайн-токены проекта вместо сырых значений из Figma
- ❌ Переписывать absolute layout в flex/grid
- ❌ Добавлять responsive классы (md:, lg:, и т.д.)
- ❌ Добавлять hover/focus эффекты которых нет в Figma
- ❌ «Улучшать» или «оптимизировать» вёрстку
- ❌ Менять data-node-id атрибуты

## ШАГ 3 — ПЕРВАЯ ВАЛИДАЦИЯ (сразу после реализации)

1. Открой реализованную страницу в браузере через MCP browser tools
2. Сделай скриншот реализации
3. Сравни со скриншотом из get_design_context

**Чек-лист сравнения:**

| # | Что проверять | Как проверять |
|---|--------------|---------------|
| 1 | Общие пропорции | Ширина и высота контейнера совпадают? |
| 2 | Позиции элементов | Каждый блок на своём месте (top/left/right)? |
| 3 | Размеры картинок | Пропорции и размер фото совпадают? |
| 4 | Цвета фонов | Серый, белый, жёлтый, чёрный — те же? |
| 5 | Цвета текста | Бренд, заголовок, цена, описание — те же? |
| 6 | Размер шрифтов | Визуально тот же размер текста? |
| 7 | Жирность шрифтов | Bold где Bold, Black где Black? |
| 8 | Uppercase | Где заглавные — там заглавные? |
| 9 | Бейджи (скидка, гарантия) | Позиция, размер, цвет, текст? |
| 10 | Кнопки (wishlist, compare, cart) | Позиция, размер, иконки? |
| 11 | Звёзды рейтинга | Количество, размер, позиция? |
| 12 | Скругления | Карточка, картинка, бейджи, кнопки? |
| 13 | Наложения (z-order) | Что поверх чего? |
| 14 | Отступы между блоками | Визуальные зазоры совпадают? |

4. **Если есть расхождения — ИСПРАВЬ ИХ.** Используй ТОЛЬКО значения из кода get_design_context.
